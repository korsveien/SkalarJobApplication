Clean Code

I try to follow the guidelines of Robert C. Martin's book "Clean Code" whenever possible. Functions are small and contain descriptive names, resulting in code that is easy to read and understand, without the necessary use of comments which -- acording to the book -- is a sign of the software developers inability to express him/her-self in code. Naturally I can write comments, and do so in projects where this is part of the convention. For solo-projects like this however, I like to avoid comments at all costs, so that it is really important that the code itself is readable.

In this project I also make full use of the potential of the javascript language. Instead of the general way to create "classes" in Javascript, I follow the module pattern, where class-constructors are functions that return objects. This allows me to hide functionality (private variables and functions), and also increases the speed of function calls (with the drawback of increased memory-usage). Following the module pattern, I also never have to use 'this', which has it's own little quirks, or remember to use the 'new' keyword when creating instances.

Web-applications often utilizes third party frameworks like JQuery. In this project I have decisively avoided such frameworks to better prove my understanding of the javascript language. All code in this project is namespaced under the 'fja' object to avoid name-collisions with eventual future code.

All javascript code in this project is available under the 'Scripts' folder.
